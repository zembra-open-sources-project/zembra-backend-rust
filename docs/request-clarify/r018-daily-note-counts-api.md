# r018 Daily Note Counts API 需求澄清

日期：2026-05-21

## 背景

前端需要获取最近 30 天每天的笔记新增数量，用于展示笔记创建趋势。该统计只面向默认 workspace 的可见笔记。

## 已确认需求

| 项目 | 结论 |
| --- | --- |
| API 路径 | 新增 `GET /notes/stats/daily-counts` |
| 时间范围 | 过去 30 天，包含服务器本地日期的今天 |
| 日期口径 | 按服务器本地时区生成日期 |
| 统计字段 | 使用 `notes.created_at` 统计每天创建的笔记数 |
| 笔记范围 | 只统计未删除、未归档笔记 |
| 空日期 | 没有笔记的日期返回 `count = 0` |
| 排序 | 按日期升序返回 |
| 请求参数 | 本轮不新增 query 参数 |
| 响应结构 | 返回 `{ "days": [{ "date": "YYYY-MM-DD", "count": N }] }` |

## 非目标

- 不支持自定义统计天数。
- 不支持按 tag、field、role 分组。
- 不统计已删除或已归档笔记。
- 不新增数据库表、缓存或后台任务。
- 不改动现有 notes 列表、recent、random 接口行为。

## 验收点

| 编号 | 验收内容 |
| --- | --- |
| A1 | `GET /notes/stats/daily-counts` 返回 `200 OK` |
| A2 | 响应包含 30 条按日期升序排列的数据 |
| A3 | 日期格式为 `YYYY-MM-DD` |
| A4 | 今天包含在最后一天 |
| A5 | 有笔记的日期返回对应数量 |
| A6 | 没有笔记的日期返回 `count = 0` |
| A7 | 已删除、已归档笔记不参与统计 |
| A8 | OpenAPI 文档包含该 path、response 和错误响应 |
