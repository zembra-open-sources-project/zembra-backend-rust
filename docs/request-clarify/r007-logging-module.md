# r007 日志模块需求澄清

日期：2026-05-04

## 需求范围

- 新增统一日志模块，服务启动时完成日志初始化。
- 日志显示级别可在 `~/.zembra.env` 中配置，默认显示级别为 `INFO`。
- 日志保存路径可在 `~/.zembra.env` 中配置，默认目录为 `logs`。
- 日志文件按天滚动保存，一天一个文件。
- 服务启动成功后打印数据库路径、base url 和支持的 API。
- 支持的 API 按 OpenAPI 当前 path 列表输出，格式为 `METHOD /path`，一行一条。

## 配置约定

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `logging.level` | `INFO` | 控制日志显示级别 |
| `logging.path` | `logs` | 控制日志文件保存目录 |

## 验收点

- 不配置 `[logging]` 时，服务使用 `INFO` 和 `logs` 默认值。
- 配置 `logging.level` 后，控制台和文件日志遵守该级别。
- 配置 `logging.path` 后，日志文件写入指定目录。
- 启动成功后能看到数据库路径、base url 和 API 列表。
